# M5.2 自动复测、统计比较与证据化归因设计

## 1. 目标

M5.1 已建立：策略任务 → 发布作品 → AI 观察 → 引用证据 → 复测实验。
M5.2 将“人工发起、单次比较”升级为“计划化、多次采样、对照校正、连续周期判断”。

## 2. 核心实验单元

一个可比较单元由以下口径共同确定：

- merchant
- question
- cohort（TARGET / CONTROL）
- collector config
- AI platform
- provider/model
- collection channel
- location/language
- prompt template version
- web-search mode

这些口径发生变化时，结果只能作为新口径或标记 context drift。

## 3. 数据流

```text
已有发布作品 / 优化任务
  → 创建复测实验
  → 绑定发布前基线
  → 配置目标问题与对照问题
  → 生成 D+3 / D+7 / D+14 / D+30 schedule points
  → Worker 领取到期点
  → question × collector × repetition 自动采集
  → M5.1 staging / DRAFT observation
  → 保存 retest samples
  → 聚合指标与置信区间
  → 目标组变化 - 对照组变化
  → 连续周期与停止策略
  → 自动归因草案
  → 人工复核
```

## 4. 指标

二元指标：

- brandMentionRate
- brandRecommendationRate
- top3Rate
- anyCitationRate
- directPublicationCitationRate

排序指标：

- meanReciprocalRank = mean(rank > 0 ? 1/rank : 0)

每个二元指标保存 numerator、denominator、rate、Wilson 95% 区间和波动率。

## 5. 波动率

MVP 使用二元样本标准差映射：

`volatilityScore = min(1, 2 * sqrt(p * (1-p)))`

- p 接近 0 或 1 时波动低；
- p 接近 0.5 时波动高；
- 该分数用于产品提示，不代表统计显著性。

## 6. 差分比较

```text
targetDelta  = targetAfter - targetBefore
controlDelta = controlAfter - controlBefore
adjustedDelta = targetDelta - controlDelta
```

没有对照组时，只能输出原始变化，归因等级上限通常为 TEMPORAL_ASSOCIATION，除非存在经核验的直接引用证据。

## 7. 归因规则

### DIRECT_CITATION

必须有作品级直接引用，并经人工复核。该等级表达“作品被引用”，不自动表达“全部提升由作品导致”。

### STRONG_ASSOCIATION

要求：

- 样本充足；
- adjusted delta 达到阈值；
- 对照组存在；
- 连续至少两个复测周期改善；
- 无模型/地区/搜索模式口径漂移；
- 无明显同期干预混杂。

### TEMPORAL_ASSOCIATION

发布后出现改善，但样本、对照、持续性或证据不足。

### INSUFFICIENT_EVIDENCE

无基线、样本不足、失败率高、口径变化、只有偶发命中或目标和对照一起变化。

## 8. 自动停止

- 达到计划末端；
- 达到调用/费用预算；
- 连续 N 个周期无变化；
- 连续 N 个周期稳定直接引用；
- 连续 Provider 失败；
- 发布作品归档/移除；
- 人工取消。

## 9. 安全与成本

- Worker 默认关闭；
- 每轮有最大领取数和最大 cell 数；
- 复用 M5.1 Provider 白名单、SSRF、密钥环境变量和 DRAFT 边界；
- 日志不输出 prompt 中的敏感商家数据和 Provider secret；
- 后续生产化可替换为队列，领域算法不依赖执行方式。
