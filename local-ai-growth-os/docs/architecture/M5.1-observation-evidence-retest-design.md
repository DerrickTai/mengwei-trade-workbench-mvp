# M5.1 自动观察、信源证据与复测闭环

## 已实现范围

M5.1 在既有 M5 正式诊断之上补充“采集—核验—证据—复测”链路。它不替代人工 App/Web 观察，也不执行浏览器自动登录、验证码绕过、自动发帖、刷评或账号池。

数据流为：消费者问题 → 采集配置 → 采集运行/暂存结果 → 可选 DRAFT 观察 → 人工核验 → M5 正式诊断；引用 URL → 规范化与公开页面元数据 → 信源证据 → 观察/采集结果关联；策略任务 → 发布作品登记 → 引用事件 → 复测实验 → 快照对比。

## 数据与安全边界

V15 新增 `geo_collector_configs`、`geo_collection_runs`、`geo_collection_results`、`geo_source_evidence`、`geo_observation_source_links`、`geo_tracked_publications`、`geo_publication_citation_events`、`geo_retest_experiments` 和 `geo_retest_runs`。所有表均带 `tenant_id` 与 `merchant_id`，服务查询以两者为范围。

采集配置只保存密钥环境变量名称；运行时才读取环境变量。密钥、请求 Authorization 头和响应中的密钥均不写数据库、日志或前端响应。当前配置仅接受 DeepSeek `api.deepseek.com` 与豆包 `ark.cn-beijing.volces.com` 的 HTTPS 官方主机。

自动采集结果首先保存为 staging 记录；即使开启自动创建观察，也只能创建 `demo=false`、`verification_status=DRAFT`、`fact_check_status=NOT_CHECKED` 的观察。M5 正式诊断仍只读取 `VERIFIED` 且非演示观察，因此 staging/DRAFT 不会进入正式分母。

## 渠道与采集

渠道区分为 `OFFICIAL_API`、`AUTHORIZED_SEARCH_API`、`USER_BROWSER_CAPTURE`、`MANUAL_APP`、`MANUAL_WEB`。官方 API collector 支持 DeepSeek Chat Completions 与豆包 Responses 响应解析；保存模型、地区、耗时、HTTP 状态、request id、原始响应摘要/结果、引用和 SHA-256。超时、429 与 5xx 有有限次数的指数退避；400、401、403 不重试。未配置密钥时产生 `SECRET_NOT_CONFIGURED` 失败结果，不创建观察。

## 信源、引用与复测

URL 会清除 tracking 参数和 fragment，并拒绝非 HTTP(S)、localhost、私网、链路本地等地址；每次重定向都会再次校验。页面抓取保存状态、标题、canonical、作者、站点、内容哈希和人工复核字段。当前仅是轻量元数据抓取，不能替代正文抓取或基础设施层 egress 防护。

发布作品引用匹配支持 `EXACT_URL`、`REDIRECTED_URL`、`CANONICAL_URL`、`CONTENT_FINGERPRINT`、`DOMAIN_ONLY`、`MANUAL_CONFIRMED`。只有前四类和人工确认计作直接引用；域名相同只表示弱关联。复测比较输出 `IMPROVED`、`DECLINED`、`STABLE`、`INSUFFICIENT_SAMPLE`、`HIGH_VOLATILITY`，归因分为 `DIRECT_CITATION`、`STRONG_ASSOCIATION`、`TEMPORAL_ASSOCIATION`、`INSUFFICIENT_EVIDENCE`。除直接引用事实外，产品不得声称内容动作导致 AI 表现变化。

## 已知限制

采集目前为同步循环；复测波动率由调用方提供；HTML 抓取未做 DNS rebinding 的连接后复核或网络 egress 隔离；不同供应商的请求字段仍应在启用前按官方文档复核。建议后续使用 stub transport 建立不调用付费模型的 CI 验收。
